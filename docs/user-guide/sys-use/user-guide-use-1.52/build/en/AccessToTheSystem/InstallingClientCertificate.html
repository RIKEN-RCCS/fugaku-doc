4.2. Client Certification Installation¶

Client certificate is used when accesing to the Fugaku website. Please install to the Web browser which accesses to the Fugaku website.

This indicates how to install the required client certificate to access to the Fugaku website of the Fugaku. This work is not required if installed the client certificate by refering to the Startup Guide.

Please prepare the followings before installing the client certificate.

  • Client certificate :”user account name .p12” file

  • Client certificate pathphrase

Client certificate

Once the account issue is compleded, the client certificate is sent via e-mail to the e-mail address you entered when applying. Please save the attached “local account name.p12” file to the device (e.g. PC) which installs the client certificate. To “local account name.p12” file, the client private key, the client certificate (a public key) and the route certificate of the client certificate issuing authority.

Client certificate pathphrase

The passphrase is sent in written form or PDF file separately from the client certificate. It will be required when installing the client certificate. Please store at the safe place.

This section describes the procedure for registering a client certificate on your PC.

Note

If you use a different Web browser than the specified Web browser, confirm the certificate management method of the Web browser yourself, and install the client certificate in the Web browser to be used.

4.2.1. Installing the certificate to Firefox (Windows)¶

This indicates how to install Firefox on Microsoft Windws. The difference may be seen depending on the version of Firefox. If the screen is different, please try with confirming the Firefox information.

  1. Start Firefox and open [Option]. Click on [Privacy and Security]‘s [Display certificate].

  1. Once the certificate manager is started, select [Your certificate] and click on [import…].

  1. Client certificate: Select “local account name.p12” file and click on [Open].

  1. Input the client passphrase to Passwordand click on [OK].

Attention

If the client certificate pass phrase is wrong it shows an error and cannot go forward.

  1. Confirm if the client certificate is registered.

  1. Select [Certificate authority certificate] and from the list, select “RIKEN R-CCS” and click on Display.

  1. Please confirm if the certificate’s Fingerprints is (SHA-1): EEED846F FC733A73 328F4561 39BDB995 D5174BBC.

  1. Next, set the password to be entered when using the client certificate. Click on Security device….

  1. Once device manager is started, select Software Security Device and click on Change password….

  1. Set any password required when using the client certificate, click on OK.

  1. After registering the password, close Device Manager. This is the end of setting the password when using the client certificate. The password set here is used when using the client certificate.

4.2.2. Installing the certificate on Firefox (Mac)¶

  1. Start Firefox, click [Preferences…]from menu.

  1. Click [View certificates…]in Privacy and security tab.

  1. Once certificate manager started, select [Your certificate] then click [Import…].

  1. Select the “local account name.p12” file saved on your computer, click [Open].

  1. Input the passphrase of the obtained client certificate to Password area, click [OK].

Attention

If the client certificate passphrase is incorrect, an error is displayed and you cannot proceed to the next screen.

  1. Confirm that the client certificate has been registered, click [OK]and close the certificate manager. This completes the client certificate installation process.

  1. Next, set the password to be entered when using the client certificate. Click Security device….

  1. Once device manager starts, select Software Security Devicethen click Change password….

  1. Set an arbitrary password required when using a client certificate, click OK.

  1. Click OK.

  1. Click OK and close devicce manager This completes the password setting procedure when using a client certificate.

4.2.3. Installing the certificate to Chrome (Windows)¶

This indicates how to install Chrome on Microsoft Windws. The difference may be seen depending on the version of Chrome. If the screen is different, please try with confirming the Chrome information.

  1. Start Chrome and open [Setting]. Click on [Privacy and Security]‘s [Security].

  1. Click on [Manage certificates].

  1. Click on [Manage imported certificates from Windows].

  1. Once certification manager is started, select [Personal] and click on [Import…].

  1. Once certificate import wizard is opened, click on [Next].

  1. Click on [Browse…].

  1. Change the file type to [Personal Information Exchange(*.pfx,*.p12)].

  1. Select “user account name.p12” file and click on [Open].

  1. After setting a file name, click on [Next].

  1. Input the client certificate passphrase to Password and check the import option [Strong the security of private key then click on [Next].

Attention

If the client certificate pass phrase is wrong it shows an error and cannot go forward.

  1. Check Automatically select certificate store based on certificate type and click on [Next].

  1. Click on [Finish].

  1. The “Import new secret exchange key” screen will be displayed continuously, click on [Set security level].

  1. Check [High] and click on [Next].

  1. Set the password and click on [Finish].

  1. Click on [OK].

  1. Click on [OK].

  1. If a security warning is issued, confirm that the thumbprint is (SHA-1): EEED846F FC733A73 328F4561 39BDB995 D5174BBC and click on [Yes].

  1. Installation of client certificate is all done.

4.2.4. Installing the certificate to Chrome (Mac)¶

This indicates how to install Chrome on Mac. At macOS, the client certificate is managed at “Key chain access”.

  1. Client certificate: Double click “user account name.p12” file. The password input screen is show first. Enter the client certificate pass phrase and click on [OK].

Attention

If the client certificate pass phrase is wrong it shows an error and cannot go forward.

  1. Open “Key chain access” screen and double click on the server certificate (RIKEN R-CCS) that issused the client certificate.

  1. Click on “Route certificate authority” - “Trust” and from the list of “When using this certificatie”, select “Always trust” and close the screen.

  1. You will be prompted for your Mac administrator username and password to reflect the change in trust settings. Input these and click on [Update setting].

  1. On the “Keychain Access” screen, hold down the Control key and click the client certificate (the local account name is shown in the name field), select [New identify preference].

  1. To “Location or Email Address:”, input “https://www.fugaku.r-ccs.riken.jp/” and click on [Add].

  1. With the same steps, register “https://api.fugaku.r-ccs.riken.jp/”.

  1. Confirm if “https://www.fugaku.r-ccs.riken.jp/” and “https://api.fugaku.r-ccs.riken.jp/” ‘s “Identify preference” is registered to “Key chain access” and close the screen. Installation is all done.
