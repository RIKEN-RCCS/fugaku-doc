Cradle CFD scFLOW 試行利用開始までの流れ (提供を停止中)¶

「富岳」上に整備されたCradle CFD scFLOWの試行利用を開始するために必要な手順は以下の通りです。

  1. 試行利用申請

  2. 利用可否・利用期間通知

  3. 試行利用開始

  4. 試行利用終了

1. 試行利用申請¶

  1. 富岳サポートサイトのお問い合わせ(https://fugaku.zendesk.com/hc/ja/requests/new) にアクセス

  2. 「各種申請」を選択

  3. 「お問い合わせ種別」から「商用アプリケーションの試行利用」を選択

  4. 「課題ID」、「件名」（例:Cradle CFD scFLOW試行利用申請）、「利用者氏名」を入力

  5. 「アプリケーション名」から「Cradle CFD scFLOW」を選択

  6. 「利用申請」に、利用申請者の情報を記入。詳細は以下「利用申請について」を参照してください

  7. 「利用条件・注意事項」を確認の上、同意チェックボックスにチェック

  8. 「送信」を押す

[利用申請について]

利用申請には、以下の内容を記入します。

  • 利用を希望するローカルアカウント

    連絡いただいたアカウントに対して、試行利用期間中に対象アプリケーションを実行可能とする設定を行います。
    利用を希望する利用者全員のローカルアカウントを記入してください。

  • 利用目的

    本試行利用の利用目的について、簡潔に記載してください。
    （例：動作確認のため。レスポンスチェックのため）

  • 試行利用希望開始日

    試行利用の期間は2週間です。試行利用の希望開始日を、yyyy/mm/dd形式で記入してください。希望開始日は、以下を考慮の上決定してください。

    • 試行利用の申請から試行利用開始まで、最短で2週間(10営業日)必要です

    • 利用不可となる保守期間や利用できるノード数が制限される大規模実行期間なども試行利用の期間としてカウントされますので、運用スケジュールをご確認ください
      運用スケジュール：https://www.fugaku.r-ccs.riken.jp/operation_schedule

    なお、諸般の事情により試行利用の開始日を変更させていただく場合があります。試行利用期間について追加の希望があればご記入ください。可能な限り考慮しますが場合によっては別途日程調整を依頼する可能性がありますのでご了承ください。

    [ 試行利用希望開始日・課題終了日 記入例 ]:

    試行利用希望開始日（第一希望）：yyyy/mm/dd ※必須
課題終了日：yyyy/mm/dd ※必須

追加希望（以下、記入例）
・第二希望日：yyyy/mm/dd 第三希望日 yyyy/mm/dd
・yyyy/mm/dd～yyyy/mm/dd の期間は除外してください
・いつでも大丈夫です

  • 課題終了日

    課題終了日（試行利用する課題の課題実施期間の終了日）を、yyyy/mm/dd 形式で記入してください。

  • 利用者情報

    以下の情報を記入してください。なお、利用者間で所属会社・組織または所属部署が異なる場合、全ての所属会社/部署の情報を提供してください。

    利用者情報
    会社・組織名：
    部署名：
    住所：※所属部署の事務所住所を記入してください。
    MSLA契約番号（既にCradle CFD scFLOWのライセンス契約されている場合）：

[ 利用条件・注意事項 ]

本試行利用はCradle CFD scFLOWの性能や使い勝手の評価をして、その後の富岳の本格利用の実施可否を判断する目的にのみ利用いただけます。利用にあたっては後述の「scFLOW評価版ライセンス規約」への同意が必要です。

また、評価後に富岳の本格利用を実施する場合、Cradle CFD scFLOWの正式ライセンスを入手し富岳で利用することが前提となるため、あらかじめソフトウェアベンダーに利用者情報を提供する必要があります。

そのため、申請時に記入頂いた以下の情報を、R-CCSが申請者の代理としてソフトウェアベンダーに提供します。

なお、R-CCSおよびソフトウェアベンダーの判断により利用が認められない場合がありますので、ご了承ください。

  • 会社・組織名

  • 部署名

  • 住所

  • MSLA契約番号

  • 利用目的

2. 利用可否・利用期間通知¶

申請内容に対して、R-CCSおよびソフトウェアベンダーで試行利用が認められるか確認を行い、可否を通知します。 試行利用が許可された場合、試行利用期間を併せて通知します。

なお、第一希望の利用期間となるとは限りません。もし、通知された利用期間に問題があった場合などは通知文の手順に従い連絡してください。

3. 試行利用開始¶

試行利用開始の通知は行いません。試行利用期間前にアプリケーションは実行できませんので、ご注意ください。利用方法については、利用可否・利用期間通知および下記リンク先を参照ください。

https://www.fugaku.r-ccs.riken.jp/doc_root/ja/user_guides/scFlowGuide/usage_scflow_ja.pdf

4. 試行利用終了¶

試行利用終了の通知は行いません。試行利用期間後にアプリケーションは実行できませんのでご注意ください。アプリケーション実行中やジョブの実行待ち中に試行利用期間が終了した場合、実行ジョブの保証はできません。
